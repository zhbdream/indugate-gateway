# 工业智能体协议网关 (Industrial Agent Gateway)

> 开源项目技术架构与功能清单 v1.0
> 
> **项目定位**：AI智能体与工业设备之间的翻译官 - 让任何Agent都能通过标准MCP协议对接真实工业设备
> 
> **核心价值**：解决工业协议碎片化问题，降低AI智能体在工业场景落地的门槛

---

## 📋 项目概述

### 项目背景
- 工业设备协议碎片化严重（OPC UA、Modbus、MQTT、S7、BACnet等）
- 每个协议对接都是重复劳动，学习成本高
- AI智能体（Agent）缺乏统一的工业设备接入标准
- 普通开发者没有真实工业设备用于测试和开发

### 解决的核心问题
1. ✅ **协议标准化**：多种工业协议 → 统一的MCP协议接口
2. ✅ **数据归一化**：不同设备的数据格式 → 统一数据模型
3. ✅ **开箱即用**：内置模拟器，零硬件依赖即可体验
4. ✅ **测试友好**：预配置公开测试服务器，一键连接

### 目标用户
- 工业智能体开发者
- 工业互联网平台服务商
- 系统集成商
- 智能制造解决方案提供商

### 核心API端点

#### 设备管理
```
GET    /api/v1/devices              # 获取设备列表
POST   /api/v1/devices              # 创建设备配置
GET    /api/v1/devices/{id}         # 获取设备详情
PUT    /api/v1/devices/{id}         # 更新设备配置
DELETE /api/v1/devices/{id}         # 删除设备
POST   /api/v1/devices/{id}/connect # 连接设备
POST   /api/v1/devices/{id}/disconnect # 断开设备
```

#### 数据操作
```
GET    /api/v1/devices/{id}/nodes           # 浏览节点
GET    /api/v1/devices/{id}/data/{nodeId}   # 读取数据
POST   /api/v1/devices/{id}/data/{nodeId}   # 写入数据
GET    /api/v1/devices/{id}/data/history    # 历史数据
```

#### 模拟器管理
```
GET    /api/v1/simulators                   # 获取模拟器列表
POST   /api/v1/simulators/{type}/start      # 启动模拟器
POST   /api/v1/simulators/{type}/stop       # 停止模拟器
PUT    /api/v1/simulators/{type}/config     # 更新模拟器配置
```

#### MCP协议
```
GET    /mcp/.well-known/mcp.json   # MCP服务发现
POST   /mcp/sse                    # SSE连接
POST   /mcp/message                # 消息交换
```

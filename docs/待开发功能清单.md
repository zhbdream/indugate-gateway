## 一、当前完成度概览

| 阶段 | 状态 | 说明 |
|------|------|------|
| Phase 1 MVP | ✅ 基本完成 | 5 协议驱动（OPC UA / Modbus / MQTT / S7 / BACnet）、3 模拟器、MCP、Web、Docker |
| Phase 2 增强 | 🚧 大部分完成 | 告警、InfluxDB 可选、ECharts、S7/BACnet 基础驱动 |
| Phase 3 企业级 | 🚧 部分完成 | JWT/RBAC/审计/设备 ACL/Prometheus/Grafana 模板 |
| 开源发布 | 🚧 待收尾 | CI 有、首版 Release 标签未打、文档/API 规范待完善 |

**技术栈说明（非 Spring Boot 全家桶）**

后端采用 **Gin + GORM + Viper + Zap** 组合式架构，依赖在 `cmd/gateway/main.go` 与 `internal/api/router.go` 中手动组装，无 Spring 式自动 DI。当前代码量可控，不算啰嗦；随着功能增多，可考虑 Wire/Fx 做依赖注入。

## 四、已知问题与技术债

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| viewer 无法使用 MCP | 中 | RBAC 拦截所有 POST，MCP 协议依赖 POST |
| OpenAPI 版本过时 | 低 | 手写 spec 未随 v0.6 更新 |
| golangci-lint 未进 CI | 低 | Makefile 有 `lint` 目标，CI 未跑 |
| 无 E2E 测试 | 低 | 仅有单元/部分集成测试 |
| BACnet COV 未经真机充分验证 | 中 | 模拟器缺失加剧此问题 |
| 完整 Docker 栈 InfluxDB 未默认启用 | 中 | 容器起了但 Gateway 未写入 |
| 主 `docker-compose.yml` 与 `deployments/docker/` 两套 | 低 | 文档需明确使用场景 |
| MCP 订阅后 Agent 需自行轮询 REST | 低 | `subscribe_data` 返回 poll_hint，无 MCP 侧事件流 |

## 七、建议开发顺序（明日可执行）

按 **1～2 天一个批次** 推进，避免摊子铺太大。

### 批次 A（半天）：修复与发布准备
1. 修复 viewer MCP 只读访问（RBAC 例外 `/mcp/message` 的 tools/call 读操作）
2. 修复 `DevicesView` S7/BACnet 协议标签
3. 完整栈 compose 补上 InfluxDB 环境变量
4. 更新 Swagger spec 至 v0.6+（至少补 auth/alerts/users）
5. 打 tag `v0.7.0` + Release Notes

### 批次 B（1 天）：模拟器补齐
1. S7 模拟器（内存 DB 区 + TCP 102 端口，可用 gos7 或简化 server）
2. BACnet 模拟器（UDP 47808，响应 ReadProperty/COV）
3. Web 模拟器页增加 S7/BACnet 卡片
4. 补充 `configs/devices.yaml` 预置 S7/BACnet 示例设备

### 批次 C（1 天）：测试与文档
1. S7/BACnet 集成测试（基于模拟器）
2. 编写 `docs/s7-bacnet-guide.md`
3. CI 增加 golangci-lint（可选）
4. 更新架构主文档 Phase 状态

### 批次 D（按需）：体验增强
1. Pinia 统一 auth + 设备状态
2. MCP Python 客户端示例脚本
3. WebSocket 实时数据（可选）
